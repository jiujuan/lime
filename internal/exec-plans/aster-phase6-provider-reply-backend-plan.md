## 目标

把 R2/R3 最后一跳固定为单一、可删除的 Aster 退场函数，并继续为后续把 provider trait object execution 迁到 `model-provider` current backend 做准备。当前事实已从旧 `provider_reply_exit_source.rs` 私有模块收敛为 `request_tool_policy/aster_reply_backend_adapter.rs::run_aster_reply_backend(...)`，且 `provider_reply_exit_source.rs` 与 `model-provider::provider_stream::source_execution` wrapper 均已删除并由治理守卫禁止恢复；Aster message / session lowering 也已收进 `AsterReplyBackendRequest::from_current(...)`，`run_aster_reply_backend(...)` 只接收一个可替换的退场 request。同时补上 Codex-style response event materializer，让非文本 delta、item lifecycle、completed 与 rate limits 先进入 Lime current projection 链。`agent-compat` 是要迁移走的 staging / compat blocker，不是 current owner；本计划不得再把 compat 命名或 `agent-compat` 方向当成完成态。

## Current / Compat / Dead

R4 native dispatch 补充分类：

- `current`：`tool-runtime::native_dispatch` 是标准 native tools 的执行事实源；`tool-runtime::native_dispatch_execution::execute_runtime_native_dispatch_tool(...)` 承接 dispatch-backed 工具接管、current permission preflight、cancel 检查、执行和结果投影；`tool-runtime::gateway_dispatch_execution` 承接 memory / image task / tool_search / MCP resource 这类 App Server gateway-backed native tools 的 current execution registration、canonical lookup、permission check、cancel fail-fast、执行和结果投影；`tool-runtime::native_overlay::runtime_native_tool_overlay_for_dispatch_name(...)` 用 current dispatch canonical lookup 反查 dispatch-backed overlay，`runtime_native_tool_registration_allowlist()` 使用 current `request_user_input` 名称；`tool-runtime::shell_execution` 承接 Bash / PowerShell 前台执行、后台启动、workspace sandbox fail-closed guard、非 full-access warning 的 current permission guard，以及 `BashTool` / `Shell` / `developer__shell` / `mcp__system__shell` / `shell_command` / `exec_command` / `local_shell_call` / `PowerShellTool` 等 legacy shell alias 的 current owner；`tool-runtime::file_read_execution` 承接 Read 文件读取；`tool-runtime::file_search_execution` 承接 Glob/Grep 文件搜索；`tool-runtime::request_user_input` 是 `request_user_input` tool surface DTO/schema/parse/validation/normalization/definition/execution owner；`tool-runtime::skill_execute` 是 Skill execution envelope / session gate / runtime contract / result metadata owner；`lime-agent::tools::execute_current_skill_tool_request(...)` 在 native hook 中先于 live shell 和 Aster registry 执行 Skill current backend；`agent-runtime::request_user_input` 是 Turn-side request_user_input runner；reply loop 只在迁移期把当前 session/turn context 适配给这些 current executor 与 request_user_input action callback。
- `compat blocker`：Aster `ToolRegistry` / `Tool` / `ToolContext` 仍是 reply loop native fallback，只允许服务 MCP source adapter 和尚未迁出的剩余 tool trait 壳；Skill 已改为 definition-only install step，不再注册 Aster `Tool` trait wrapper，模型可见 definition 由 current `tool-runtime::skill_gate` / `Agent::list_tools` 注入，执行由 native hook 先于 registry 接管；MCP 已不再挂入 Aster registry，但 Aster extension manager / MCP client bridge 仍是 source adapter blocker；真实 workspace sandbox backend 仍未实现，`agent-compat::sandbox::SandboxConfig` 只作为 Aster `ToolContext` 的最小 marker DTO 暂留，不能承接 sandbox execution；标准 dispatch-backed tools、gateway-backed native tools、foreground/background shell、legacy shell alias、shell warning guard、workspace sandbox guard、Read、Glob、Grep、request_user_input 和 native-hook Skill 执行不得再优先走 `registry.execute(...)`。
- `dead / guarded`：标准 native tools、gateway-backed native tools、foreground/background/workspace-sandbox/warning shell、legacy shell alias、Read、Glob、Grep、request_user_input 或 native-hook Skill 重新优先走 Aster `registry.execute(...)`，恢复 `agent-compat/src/tools/ask.rs` / Aster `AskTool` 注册壳，恢复 `agent-compat/src/tools/{bash.rs,powershell_tool.rs,task.rs}`、Aster shell alias registry ownership、Aster empty sandbox executor / output buffer / `SandboxType`，在 `agent-compat` 恢复本地 native tool-name overlay 表 / gateway execution owner / file read owner / search owner / Skill execution owner，公开 re-export `BashTool` / `PowerShellTool` / `ReadTool` / `GlobTool` / `GrepTool` / `AskTool` 具体实现类型，或把 current permission / approval preflight 写成 Aster session cache / hook / pending map。

- `current`：`agent-runtime` provider handle 校验、provider wire policy、session metadata 注入和 `RuntimeReplyBackendRunPath` 分派 contract，`model-provider` provider handle / stream request、provider response event/item DTO、provider response content contract、provider response context header extraction、provider sampling request/mode、tool-input delta event policy、lead/worker model-change policy、provider stream notification envelope / text classification、progress / milestone state、provider failure logging / telemetry category classification、provider trace failure projection / trace DTO alias、plaintext tool-use parser / stream progress、image input policy 与 empty-first-content retry policy，`tool-runtime::turn_tool_surface` 的 turn tool surface / scope policy、tool exposure / resource gate / registration gate，`tool-runtime::tool_call_surface` 的工具调用名称和参数 normalization，`tool-runtime::tool_result_projection` 的 native tool result -> MCP `CallToolResult` 投影和 `tool_surface_updated` 判定，`agent-runtime::reply_stream::RuntimeReplyResponseMaterializer`。
- `transitional current adapter`：`request_tool_policy/agent_reply_stream.rs::runtime_agent_events_from_response_event(...)`，暂时把 response projection 转进现有 `RuntimeAgentEvent` / timeline item 主链；`request_tool_policy/response_event_adapter.rs` 持有 `RuntimeAgentEvent` -> `RuntimeReplyResponseEvent` 过渡映射；`request_tool_policy/aster_reply_stream_adapter.rs::AsterReplyStreamProjector` 只作为 Aster source adapter，把 Aster message/direct tool-input delta 前移成 current `RuntimeReplyResponseEvent`；`request_tool_policy/aster_reply_backend_adapter.rs::AsterReplyBackendRequest` 只作为 Aster 最后一跳的 request lowering 退场壳，后续 current provider backend 接管时整体删除。
- `compat blocker`：`request_tool_policy/aster_reply_backend_adapter.rs::run_aster_reply_backend(...)`，只允许作为删除前唯一 Aster provider reply 最后一跳；其内部仍调用 `Agent::reply_with_provider(...)`，退出条件是 provider trait object / provider stream execution 迁到 `model-provider` current backend 后删除该函数中的 Aster lowering / provider trait object / projector。
- `dead / guarded`：`compat_provider_reply_backend.rs`、`CompatProviderReplyBackend`、`CompatReplySource*`、`run_compat_provider_reply_backend(...)`、`Aster*ReplySource`，恢复 `agent-compat::Agent::reply(...)` wrapper，把 provider source execution wrapper 放回 `agent-runtime` / `agent-compat` / `provider_stream.rs` 主文件，把 `agent-compat` 当作 current owner 或让 `agent-compat` 新增 current owner 依赖的任何改动，response event 只消费 text delta、吞掉 tool input / reasoning / item / completed / rate limits 的旧形状，在 `agent-compat/src/agents/agent.rs` 恢复 provider response text / notification / tool-input delta 直接规则，恢复 `agent-compat` 本地 `ProviderTraceStage` / `ProviderTraceEvent` DTO，恢复 `event_converter` provider trace stage 映射 / 字段复制，或恢复 native tool result projection helper。

## 本轮任务

R4 native dispatch 补充任务：

- 标准 native tools 必须在 live-execution hook 之后先走 `tool-runtime` dispatch-backed overlay lookup、current permission preflight 和 current dispatcher；Aster registry 只作为未迁工具 fallback。
- Bash / PowerShell 前台执行与 background 启动必须在 live-execution hook 后先走 `tool-runtime::shell_execution`，由 current owner 处理参数、permission/preflight、embedded execution、output decode、background output file 和 result projection；workspace sandbox shell 不得再 fallback 到 Aster sandbox stub，真实 backend 未实现前由 current guard fail-closed；非 full-access warning 不得再 fallback 到 Aster registry，真实 HITL 后续只允许接 App Server RuntimeCore pending action / `agentSession/action/respond` 主链，不在 `agent-compat` 内新增 approval cache。
- Read / Glob / Grep 前台执行必须在 live-execution hook 和 shell/collab current executor 之后、native dispatch 与 Aster registry fallback 之前先走 `tool-runtime::{file_read_execution,file_search_execution}`；`agent-compat` 只能传递 Aster `ToolContext` 的 cwd / cancel token，不得恢复文件读取 / 搜索执行 owner。
- Gateway-backed native tools 必须在 Read / Glob / Grep 之后、标准 native dispatch 与 Aster registry fallback 之前先走 `tool-runtime::gateway_dispatch_execution`；`lime-agent` 注册 memory / image task / tool_search / MCP resource 时只携带 current execution registration，`agent-compat` 只能保存 registry 和适配 Aster `ToolContext`，不得重新承接 gateway DTO、permission 或 result projection。
- Skill 执行必须在 native hook 中先走 `lime-agent::tools::execute_current_skill_tool_request(...)` + `tool-runtime::skill_execute`；Skill 模型可见 definition 必须由 current `tool-runtime::skill_gate::skill_tool_definition()` 注入，`RuntimeNativeToolInstallStep::registers_aster_tool()` 对 Skill 必须为 false，不得恢复 `LimeSkillTool`、Aster `Tool` trait wrapper、session gate、runtime contract、result metadata 或执行 owner。
- `request_user_input` 的 tool surface contract、definition 和 reply-loop execution helper 必须归属 `tool-runtime::request_user_input`；`agent-runtime::request_user_input` 只能做 Turn-side runner，`agent-compat` 只能保存显式注入的 callback 配置并调用 current executor，不能再提供隐式默认 callback；Aster `AskTool` 文件和 registry 注册不得恢复。
- `agent-compat/src/agents/agent.rs` 中新增/保留的 helper 只能做当前 session/turn context 适配、错误投影和 dispatch 调用编排；不得在 `agent-compat` 内复制工具名表、permission policy 或 approval cache。
- 治理测试必须断言 `runtime_native_tool_overlay_for_dispatch_name(...)` 存在、reply loop 消费 `runtime_native_dispatch_handle`，且 `execute_runtime_native_dispatch_tool(...)` 的调用顺序早于 fallback `registry.execute(...)`。
- 治理测试必须断言 `tool-runtime::gateway_dispatch_execution` 存在、`lime-agent` gateway bridge 为每个 gateway-backed surface 注册 current execution registration，且 `execute_runtime_gateway_tool(...)` 的调用顺序早于标准 native dispatch 和 fallback `registry.execute(...)`。
- 治理测试必须断言 Aster AskTool 文件/注册不恢复，且 `tool-runtime::request_user_input` 持有 DTO/schema/parse/validation/normalization/definition/execution owner。

1. 源码不得恢复 `provider_reply_exit_source.rs` 或 `mod provider_reply_exit_source;`；Aster provider 最后一跳只允许集中在 `aster_reply_backend_adapter.rs::run_aster_reply_backend(...)`。
2. `run_aster_reply_backend(...)` 只能保留删除前必要的 Aster lowering、`Agent::reply_with_provider(...)` 和 `project_aster_reply_stream(...)`；不得恢复 `ReplyExitSource`、`ProviderReplyExitSource`、provider source backend trait/wrapper 或 `RuntimeReplyProviderExecution*` 旧骨架。
3. `aster_reply_backend_adapter.rs` 不得直接暴露 `Arc<dyn Provider>` / `dyn Provider` 签名；provider trait object 必须被 `credential_bridge::CompatReplyProvider` 包住，直到 current provider backend 替换该最后一跳。
4. 治理测试必须同时断言 `provider_reply_exit_source.rs`、`provider_stream/source_execution.rs`、`compat_provider_reply_backend.rs` 不存在，旧 `Compat*` / `Aster*ReplySource` / `ProviderReplyExit*` / `RuntimeReplyProviderExecution*` 类型和函数名不回流。
5. `lime-rs/crates/agent-compat/**` 只允许发生迁出、删除或减少依赖的改动；不得新增 `agent-compat` 到 current owner 的依赖，也不得把 `agent-compat` 当作 owner 继续补 reply loop / provider / tool / session 逻辑。`agent-compat` 现存 `document-preview` / `model-provider` / `tool-runtime` 本地依赖只是 burn-down allowlist，退出条件是迁出对应调用并删除。
6. `RuntimeReplyResponseMaterializer` 必须覆盖 `OutputItemAdded`、`OutputItemDone`、`ToolCallInputDelta`、`ReasoningDelta`、`Completed`、`RateLimits` 的 current projection，并由 `agent_reply_stream.rs` 消费到现有 runtime event / timeline item 主链。
7. `AsterReplyStreamProjector` 必须把 Aster Message text/thinking/tool-input delta、direct `AgentEvent::ToolInputDelta` 和可表达的 item lifecycle 转为 `RuntimeReplyResponseEvent`，不得继续把这些非文本/工具参数流或 provider response item 直接输出为 legacy-shaped runtime delta。
8. 本轮不宣称删除 `Agent::reply_with_provider(...)`；该最后一跳仍是 R2/R3 compat blocker。
