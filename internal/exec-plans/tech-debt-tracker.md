# 技术债追踪

本文件记录需要持续、小额偿还的技术债，避免把问题堆到一次性大清理。

## 记录规则

1. 每条技术债都要写清具体代码面、影响和下一小步
2. 能回挂路线图主线的，优先回挂路线图；不能回挂的，登记到这里
3. 状态至少区分 `待处理`、`进行中`、`已完成`、`放弃`

## CCD-012 执行规则

`CCD-012` 是 production command current 迁移期间的常驻治理项，不是一次性文档备注。后续每个命令组收口时，如果发现 `src/lib/dev-bridge/commandPolicy.ts`、`src/lib/dev-bridge/mockPriorityCommands.ts`、desktop-host mock、旧 smoke、catalog 或 retired guard 里仍保留旧命令字符串，必须先把它分类为 `compat / deprecated`、`dead / retired guard-only` 或 `test-only`。

只在当前命令组内短期阻塞的 residual，登记到对应执行计划即可。跨命令组、跨文件或会长期存在的 legacy policy / mock residual，必须同步回挂到 `CCD-012`：写清命令名、阻塞文件、当前分类、退出条件和下一次验证入口。`src/lib/dev-bridge/safeInvoke.ts`、HTTP client、`app_server_handle_json_lines`、事件监听和可用性探测继续属于 `current` renderer bridge；没有新的 renderer bridge 事实源前，不允许把整目录删除当成治理完成。

改变 `src/lib/dev-bridge/**` 分类、退场条件或后续治理口径时，同步检查 `AGENTS.md`、`docs/README.md`、`internal/README.md`、`internal/aiprompts/README.md`、`internal/aiprompts/commands.md`、`internal/aiprompts/governance.md`、`internal/aiprompts/quality-workflow.md`、`internal/aiprompts/parallel-agent-collaboration.md`、`internal/roadmap/appserver/README.md`、`internal/exec-plans/README.md` 与 `internal/exec-plans/production-command-current-migration-plan.md`，不能只改单个计划或只留在聊天 / handoff。
