## S1

- `ThreadStatus`、`TurnStatus`、`ItemKind`、`ItemStatus`。
- `Thread`、`Turn`、`ThreadItem` 和 `ThreadHistoryChange`。
- Item 使用 `(thread_id, turn_id, item_id)` 稳定身份，并同时保留 `sequence` 与 `ordinal`。

- App Server item lifecycle notification 从现有 event payload 投影可选 `ordinal`，同步重新生成 JSON schema fixtures。

## S2

`ProjectionStore` 当前补丁：

- 同一 session、同一 sequence 的同 event ID 重放幂等返回 0。
- event ID 在不同 session 或 sequence 重用时显式返回 identity collision，事务不改变 read model。
- session watermark 丢弃 `sequence <= last_sequence` 的 stale event。
- `apply_events`/`repair_session` 返回实际 materialize 数量。

新增 stale、duplicate、跨 session collision 回归测试。定向 app-server Cargo 测试因同一 S3 lowering 编译基线阻塞；rustfmt 和 `git diff --check` 已通过。
