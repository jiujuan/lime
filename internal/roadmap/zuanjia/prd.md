# 专家功能 PRD

更新时间：2026-05-15

## 一句话目标

在 Lime 中提供一个“专家广场”，让用户像选择专业顾问一样选择 Agent：先通过榜单、分类和搜索发现专家，再查看专家详情，添加后进入带专家身份、记忆、技能和工作流的对话。

## 背景

当前 Lime 已有 Agent Chat、Skill Catalog、ServiceSkill、Memory、Workflow 和 Plugin 平台能力，但用户仍需要理解底层能力名、工具名或命令入口。专家功能把这些能力重新包装成面向任务和角色的入口，让用户先回答“我需要哪位专家帮我”，再由系统完成 prompt、技能和工作流绑定。

截图参考的关键体验包括：

- 专家广场：顶部推荐榜、热门榜、上新榜，下面是分类筛选、搜索和专家卡片网格。
- 专家详情：打开后展示专家产出示例、可执行任务、技能标签、使用量、点赞、添加和分享。
- 专家对话：添加后进入普通聊天页，但右侧显示专家信息、简介、记忆、日记、技能和工作流程。

## 目标用户

| 用户 | 需求 |
|---|---|
| 普通创作者 | 不想配置工具和 prompt，希望直接选择“短视频脚本专家”“资料整理专家”。 |
| 运营 / 增长人员 | 需要固定专家长期跟进趋势、竞品、投放和复盘。 |
| 团队负责人 | 希望给团队预置可信专家，降低新人使用门槛。 |
| 高阶用户 | 需要知道专家背后的技能、记忆和工作流，便于调试和定制。 |

## 产品形态

### 1. 专家广场

页面类型是卡片型列表工作台。结构为：

1. 页面标题：`专家广场`。
2. 榜单区：`为你推荐`、`热门精选`、`最近上新` 三块轻量榜单。
3. 筛选区：分类 tabs + 搜索框。
4. 专家卡片网格：头像、标题、摘要、来源、使用量、点赞、添加状态。

默认分类：

```text
全部 / 营销部 / 办公协同部 / 生活服务部 / 金融部 / 娱乐社交部 / 学术教育部 / 工程部 / 法务部 / 销售部 / 供应链部 / 产品部 / 人力资源部
```

卡片点击进入详情浮层；卡片上的主按钮只负责添加或开始对话，避免“选中但无结果”的假交互。

### 2. 专家详情

详情以大浮层承载，不跳出当前广场上下文。详情内容分左右两区：

- 主内容：专家名称、示例任务、产出样例、适用场景、工作流说明。
- 侧栏：头像、标题、使用量、点赞、添加、分享、简介、技能标签。

首期详情中的产出样例可以来自专家 release metadata 的 `showcase` 字段；后续再接真实精选案例。

### 3. 添加专家

添加专家写入客户端 install overlay，不修改云端目录本体。

添加后：

- 广场卡片显示已添加或可直接开始对话。
- 对话侧边栏出现该专家。
- 新建专家会话时自动带入 persona、starter prompts、memory template 和 skill/workflow refs。

### 4. 专家对话

专家对话仍是现有 Agent Chat 的一种入口，不新增并行聊天页面。

首屏显示：

- 专家头像和欢迎语。
- 1-3 个 starter prompts。
- 输入框继续使用现有 Agent input bar。

右侧专家信息面板显示：

- 专家信息：头像、标题、摘要、编辑入口。
- 简介：Who I Am、How I Talk、Boundaries。
- 记忆：当前项目关注、经验积累。
- 日记：首期可为空态。
- 技能：绑定的 Skill / ServiceSkill 标签。
- 工作流程：趋势分析、内容策划、挑战赛设计、数据优化等步骤。

## ExpertProfile 投影字段

`ExpertProfile` 是客户端消费的专家产品投影，不是运行时执行格式。

| 字段 | 说明 |
|---|---|
| `id` | 稳定专家 ID。 |
| `slug` | URL / deep link 友好的唯一短名。 |
| `title` | 专家名称。 |
| `summary` | 卡片摘要。 |
| `avatar` | 头像 URL 或内置 token。 |
| `category` | 分类 key。 |
| `tags` | 能力标签。 |
| `source` | `cloud_catalog` / `seeded_fallback` / `local_custom`。 |
| `stats` | 使用量、点赞、热度、上新时间。 |
| `ranking` | 推荐、热门、上新或分类榜单位置。 |
| `personaRef` | 版本化 persona 引用。 |
| `memoryTemplateRef` | 可选记忆模板引用。 |
| `skillRefs` | 绑定技能引用。 |
| `workflowRefs` | 绑定工作流引用。 |
| `promptStarters` | 首屏建议任务。 |
| `showcase` | 详情页展示的样例产出。 |
| `readiness` | 模型、技能、浏览器、项目等依赖。 |

## 非目标

1. 不在首版做付费分账、第三方创作者后台或公开插件市场。
2. 不允许 LimeCore 默认执行专家对话。
3. 不把专家实现成第二套 Skill / Scene / Agent runtime。
4. 不把用户私有对话、私有记忆或 workspace 文件上传到公共专家目录。
5. 不先做复杂审核后台；首期只规划 release metadata 和租户可见性。

## MVP 验收

- 用户能在专家广场查看三块榜单、分类、搜索和专家卡片。
- 用户能打开专家详情，看到简介、样例、技能标签和添加按钮。
- 用户能添加专家，并从专家启动一个普通 Agent Runtime 会话。
- 会话首轮能带入专家 persona 和 starter prompt。
- 右侧专家信息面板能展示简介、记忆、技能和工作流程。
- 离线时可展示上次缓存目录或 seeded fallback。
