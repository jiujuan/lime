# App Server 前端对接矩阵

> 状态：current planning source
> 更新时间：2026-06-06
> 作用：把 App Server JSON-RPC、Electron Desktop Host IPC、前端 API 网关、mock 和真实 UI caller 对齐到同一张表，避免继续出现“后端接口存在，但前端只接 mock / compat”的半闭环。

## 1. 当前结论

当前正确架构固定为：

`Frontend -> Electron Desktop Host bridge -> App Server JSON-RPC -> RuntimeCore / backend`

Electron main 是 Desktop Host bridge，不是后端适配层，也不是第二套后端；它只负责接收 renderer IPC / legacy command facade、转换 App Server JSON-RPC method、管理窗口 / 托盘 / Dock / updater / sidecar 生命周期，以及做少量参数和返回值投影。真正的后端事实源只有一个：App Server JSON-RPC。

如果某条路径使用 in-process App Server 或 typed channel，它也只是去掉进程边界，必须继续保持 App Server JSON-RPC result envelope 和 `agentSession/event` 事件合同；不得引入第二响应合同、第二 read model 或 Electron / App Server 业务层。

当前 App Server 已经具备 protocol / Rust processor / TypeScript client / Electron sidecar 通道的骨架；本轮开始把前端主 Agent lifecycle 切到上述 current 主链。

事实口径：

1. `packages/app-server-client/src/index.ts` 已列出并实现 App Server method builder / connection helper，覆盖 `agentSession/*`、workspace、skill、model、artifact、evidence 等方法。
2. `packages/app-server-client/src/protocol.ts` 是 renderer-safe 协议事实源，只放 JSON-RPC method 常量、wire DTO、method catalog、schema manifest 类型与 encode/decode / type guard；`index.ts` 只在此基础上增加 Node sidecar、stdio connection、release manifest 和 lifecycle helper。
3. `electron/appServerHost.ts` 已能启动 sidecar，并提供 `request(...)`、`handleJsonLines(...)`、`drainEvents(...)`。
4. `electron/hostCommands.ts` 已把部分 legacy 前端命令投影到 App Server method，例如 session list/read、workspace、skill、model 和 capability inventory。
5. `electron/ipcChannels.ts` 只把 `app_server_handle_json_lines` / `app_server_drain_events` 作为通用 App Server JSON-RPC IPC 暴露给 renderer。
6. `src/lib/api/appServer.ts` 是 renderer 侧 JSON-RPC client gateway，复用 `packages/app-server-client/src/protocol.ts` 的协议类型 / 常量 / helper，底层只通过 `app_server_handle_json_lines` / `app_server_drain_events` 进入 Electron Desktop Host IPC。

8. `src/lib/desktop-host/agentRuntimeMocks.ts` 已移除 `app_server_handle_json_lines` / `app_server_drain_events` 与 submit / cancel / respond lifecycle 默认成功值；后续如需测试夹具必须显式注入，不得恢复 default mock。

## 2. 状态定义

| 状态           | 含义                                                                                                                                                   |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `wired`        | UI 主路径经前端 gateway -> Electron Desktop Host bridge -> App Server JSON-RPC method 进入真实 sidecar 或 in-process App Server，并有契约或 GUI 证据。 |
| `bridge-wired` | UI 仍调用 legacy 命令名，但 Electron Desktop Host bridge 已把它同步投影到 App Server current method。                                                  |
| `client-only`  | protocol / TS client 存在，但没有真实 UI caller。                                                                                                      |
| `backend-only` | Rust method 存在，但 Electron / 前端 gateway 未接。                                                                                                    |
| `partial`      | 主链已有 API / hook 或局部 GUI 证据，但缺完整业务 E2E、完整投影或旧入口退场证据。                                                                      |
| `mock-only`    | 只有测试夹具 / mock 证据，没有真实 host / App Server 证据；生产交付一律视为未完成或阻塞，不得作为可用闭环。                                            |
| `blocked`      | 当前被已知编译、runtime 或并行写集阻塞。                                                                                                               |

## 5. 第一竖切交付标准

第一竖切只认 `Agent turn lifecycle`，不再用“接口已存在”作为完成依据。

必须同时满足：

1. 前端主发送路径必须经 `src/lib/api/appServer.ts -> app_server_handle_json_lines -> agentSession/turn/start`，不能直接把 `agent_runtime_submit_turn` 当事实源。
2. cancel / interrupt 路径必须经 `agentSession/turn/cancel`，不能只接 submit。
3. action respond 路径必须经 `agentSession/action/respond`，不能继续只由 mock 返回 `{}`。
4. App Server event 至少能被前端 API 层消费为一次 read model 刷新或 timeline append；当前已由 `agentRuntimeEvents` 本地 fanout + `useAgentRuntimeSyncEffects` 当前 turn event 订阅触发 `refreshSessionDetail`。
5. session create / list / read 必须经 App Server session facade，不能继续把 `agent_runtime_create_session`、`agent_runtime_list_sessions`、`agent_runtime_get_session` 当成前端事实源。
6. `src/lib/desktop-host/agentRuntimeMocks.ts` 不得恢复 `app_server_handle_json_lines` / `app_server_drain_events` 或 submit / cancel / respond lifecycle 默认成功值；测试必须显式注入 fake。
7. Contract guard 必须能防止前端 lifecycle 回流到 `agent_runtime_submit_turn`、`agent_runtime_interrupt_turn`、`agent_runtime_respond_action`。
8. GUI 证据以 Electron 为准：`npm run smoke:electron` 已证明 Electron renderer + Desktop Host bridge + App Server sidecar initialize 可用；Rust/TS 单测不能单独证明可交付。
9. 第一竖切已有前端业务闭环测试覆盖 App Server turn notification 触发 read model refresh；尚需补真实 GUI 发送 / event refresh / thread read refresh 的业务 E2E，不能把 initialize smoke 或 hook 测试误判成完整聊天闭环。
10. `agent_runtime_submit_turn` / `agent_runtime_interrupt_turn` / `agent_runtime_respond_action`、desktop-host mock、mock-only 测试或 legacy facade 投影只能作为负向回归扫描对象，不能作为 Agent turn lifecycle 的完成证据。
11. 完整 Claw streaming GUI E2E 必须显式传入 `--allow-live-provider` 或 `LIME_ALLOW_LIVE_PROVIDER_SMOKE=1`；未授权时 smoke 必须在 `wait-health`、`prepare-runtime`、`aster_agent_init`、`get_api_key_providers`、`test_api_key_provider_chat` 之前 fail closed，不能用 mock、无模型探针或 Provider 自动探测替代真实业务闭环。
