## 1. 账本口径

本文件把 Codex 里的具体测试文件和测试函数索引成 Lime 必须补齐的 Clawstream 场景。它不是泛泛参考清单，而是后续补 fixture、删旧实现和评审回归时的最小账本。

每条场景必须最终落到三层中的至少两层；触达用户主路径、history hydrate、MCP、Skills、Multi-Agent、artifact 的场景必须三层都具备：

| 层级                      | Lime 目标                                                                                                                               |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| Event / Item oracle       | `packages/agent-ui-contracts` 或 `packages/agent-runtime-projection` 能 replay 出标准 Item / ContentPart / read model                   |
| Projection / DOM oracle   | `MessageList`、`StreamingRenderer`、Inputbar、right surface、artifact / markdown / diff snapshot 有结构化断言                           |
| Electron fixture evidence | `scripts/agent-runtime/claw-chat-current-fixture-smoke.mjs --scenario ...` 或聚合 smoke 产生真实 Electron + App Server sidecar evidence |

状态说明：

| 状态               | 含义                                                                          |
| ------------------ | ----------------------------------------------------------------------------- |
| `covered-electron` | 已有 Electron fixture，但还缺 item-level / projection oracle 时仍不能删旧实现 |
| `partial`          | 有局部 unit / DOM / fixture，缺至少一层标准化证据                             |
| `partial+guard`    | 有局部 oracle，且已补防回流 guard / regression test，但仍缺至少一层标准化证据 |
| `missing`          | Lime 还没有标准 Clawstream 场景                                               |
| `guard-needed`     | 需要先补防回流守卫，再删除旧 helper / fallback                                |
