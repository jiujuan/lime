## 1. 目标拆解

整体目标可落成以下交付标准：

1. **边界清晰**：Lime Agent 负责运行事实、工具、权限、证据和 Claw 能力复用；App 负责业务 UI、表单、项目状态、资料版本、产物展示和人工确认。
2. **App 内闭环**：内容工厂等 App 不跳回 Lime 通用 Chat；用户在 App 页面内发起任务、看进度、补上下文、确认、取消、重试和保存结果。
3. **完整 AgentRuntime**：App 的 AI 任务走 `agent_app_runtime_* -> agent_runtime_submit_turn` current 主链，而不是只拿 `LIME_GATEWAY_*` 低阶模型 API。
4. **Claw 能力复用**：搜索、图片、研报、PDF、总结等能力通过 capability hints / catalog 复用现有 Claw skill launch metadata，不复制 `*_skill_launch.rs`。
5. **产物和证据落库**：任务结果、人工确认、运行事件、artifact 和 evidence 能回投到 App 页面与 `lime.storage / lime.artifacts / lime.evidence`。
6. **可恢复**：刷新或 Host 重建后，App 仍可凭 `taskId` 找回 `sessionId / turnId` 并继续读取任务或提交 Host response。
7. **可验证**：命令四侧、前端 Host Bridge、Rust facade、内容工厂包、文档事实源都有对应测试或检查。

## 5. 完成判定

按“Agent App 能在 App 内调用完整 Lime AgentRuntime，并完成内容工厂主流程第一刀”的基础 runtime 口径，当前已经达到 **current MVP**。真实 Lime 宿主 iframe Host Bridge 已有端到端证据：内容工厂在 Lime 页面内可发现 bridge、发起任务、通过 task subscription 显示进度，使用隐藏 `agent-app-runtime-*` session，不跳回通用 Chat，并把 AgentRuntime 产出的 `contentFactoryWorkspacePatch` 物化成 App 页面内的文案、脚本和图片需求。
