Use case: ui-mockup
Asset type: Lime 桌面端产品原型图
Primary request: 基于用户已有截图的 Lime 桌面端产品界面，生成一张「应用中心」一级页面高保真产品原型图。

产品定位：给普通用户使用，不是开发者诊断页。用户打开页面后，应在 5 秒内知道有哪些应用、应用来自云端还是本地、是否已安装、是否有更新、是否需要授权、下一步该点哪里。

页面主题：应用中心
页面类型：桌面端一级页面，左侧主导航 + 顶部工具栏 + 中间分页应用列表 + 右侧应用详情面板。

必须保留的 Lime 壳：
- 保留参考截图风格的 macOS 窗口顶部红黄绿圆点。
- 左侧固定导航栏，保留 Lime 标识、搜索任务、新建任务、专家、Skills、内容工厂、项目资料、最近对话、归档、底部用户区。
- 当前选中项为「内容工厂」或「应用中心」均可，但不要重新设计成另一套导航体系。
- 桌面端 16:10 大画布比例，不要移动端，不要手机壳。

主内容布局要求：
1. 顶部标题为「应用中心」。
2. 副标题只写一句：「管理可以在 Lime 里打开的应用。」不要长篇说明。
3. 顶部右侧有搜索框「搜索应用名称或用途」、按钮「安装本地应用」、按钮「刷新」。
4. 标题下方是一排状态筛选胶囊：「全部 57」「已安装 3」「可安装 51」「需要处理 2」。
5. 第二排轻量筛选：「来源：全部 / 云端 / 本地」「状态：全部 / 有更新 / 需要授权」「排序：最近更新」。
6. 主区左右分栏：左侧 62% 是应用列表，右侧 38% 是当前选中应用详情。
7. 左侧应用列表是分页列表，不是卡片墙。每一行要清晰、留白足、可扫描。
8. 每个应用列表项必须显示：应用名、一句用途、来源标签、状态标签、版本摘要、主动作按钮。
9. 列表底部显示分页：「第 1 / 6 页」「上一页」「下一页」。
10. 右侧详情面板展示当前选中的「内容工厂」，只展示普通用户能理解的信息和下一步动作。

左侧列表必须出现这些应用行，中文必须清楚：
- 「内容工厂」标签「云端」「已安装」「有更新」；说明「用项目资料生成草稿、脚本、图片需求和交付包。」；版本「当前 0.1.7 · 云端 0.3.0」；按钮「打开」「更新」。这一行为选中态，浅青绿底或边框。
- 「数据分析」标签「云端」「可安装」；说明「整理表格、指标和业务数据，生成分析摘要。」；按钮「安装」。
- 「本地写作助手」标签「本地」「已安装」；说明「从本机文件夹安装，只在这台电脑可用。」；按钮「打开」。
- 「社媒发布助手」标签「云端」「需要授权」；说明「需要企业授权后安装。」；按钮「激活」。

右侧详情必须包含：
- 标题「内容工厂」
- 标签「云端」「已安装」「有更新」
- 主按钮「打开应用」
- 次按钮「更新到云端版」
- 区块标题「需要处理」
- 文案「云端新版需要企业授权码。」
- 输入框 placeholder「输入授权码」
- 按钮「激活」
- 区块标题「常用入口」
- 四个入口按钮：「项目总览」「资料整理」「内容生成」「交付复盘」
- 区块标题「来源与版本」
- 文案「已安装：本地 · 0.1.7」「云端：0.3.0，可更新」
- 底部折叠按钮「更多信息」

视觉风格：
- 符合 Lime 桌面 GUI：轻盈、清晰、专业，浅色背景，实体白色 / 浅米灰内容面板，清晰边框，浅阴影。
- 信息优先，减少废话，不要密集小字。
- 主按钮使用深海军蓝 / 深石板蓝实心。
- 选中态使用浅青绿底和深青绿文字。
- 来源标签：云端用浅天蓝，本地用灰绿。
- 状态标签：已安装用柔和绿色，有更新用天蓝，需要授权用琥珀色，可安装用浅灰绿。
- 右侧详情比左侧列表更安静，不能抢主操作。

严格禁止：
- 不要移动端，不要手机预览，不要网页营销页。
- 不要英文假文案，不要乱码，不要品牌口号。
- 默认 UI 不要出现这些工程词：Agent Apps、Cloud Catalog、local_folder、Package hash、Entries、degraded、manifest、runtime、workflow key、APP.md、sha256、page:dashboard、workflow:knowledge_builder。
- 不要展示完整本机路径、hash、manifest、权限 JSON、运行日志。

输出质量：一张清晰可读的桌面端高保真产品原型截图，中文文字尽可能准确，大模块布局和信息层级要明显。
