审计目标：把“帮我完成整个目标实现”拆成可验证交付物，逐项映射到当前代码、文档、命令和 evidence。只有本表全部进入 `PASS`，且无发布级缺口时，active goal 才能判定完成。

## 1. 目标重述

本目标不是只回答“AgentAPP 能不能独立安装”，而是要形成一条可开发、可验证、可继续产品化的主链：

1. `agentapp` 标准升级到 v0.8，明确 standalone / runtime-backed / in-Lime 安装模式。
2. Lime 客户端在 `docs/roadmap/agentapp/v2` 下形成 PRD、架构、接口契约、代码规划和实施计划，用于指导后续开发。
3. Lime current Agent App 主链支持 v0.8 install contract、RuntimeProfile、Shell descriptor、packager descriptor 和 standalone dev shell。
4. Content Factory 能以 standalone shell 启动，并完成最小 Agent / Artifact / Evidence / workspace patch / 页面物化业务闭环。
5. 模块化、扩展升级、隔离、解耦不是散文承诺，而是有代码边界和结构测试守卫。
6. macOS 身份策略明确：standalone App 使用独立 Bundle ID / App ID；Lime 官方分发可复用同一 Team 的 Developer ID Application 证书，`.pkg` 另用 Developer ID Installer 证书。
7. 发布级产品化仍需要单独闭环：产品化 App Shell、installer、signing、notarization、updater、卸载 delete-data 和 release evidence。

## 3. 当前结论

### 已完成到 MVP / P4

- 标准侧：`agentapp` 已发布 v0.8.0，main 与 tag 对齐。
- 文档侧：v2 PRD、架构、接口契约、代码规划、实施计划和本审计已落盘。
- 工程侧：install contract、RuntimeProfile、Shell descriptor、packaging descriptor、App Shell dev adapter、import boundary guard 已进入 current 主链。
- 证据侧：Content Factory standalone shell、host 合并入口、run-scenarios 业务闭环和正式 P4 evidence pack 已通过。

### 尚未完成到发布级产品

- P5 产品化 App Shell 壳已完成纯逻辑 chrome descriptor，并已把 chrome policy 作为 dev shell / mock 启动证据返回；真实原生 adapter 已完成 close event 首刀、per-app native menu 事件 handler、`lime-agent-*://open` deep link 路由、native shell registration plan、standalone Tauri config materializer、deterministic config/env write plan 和受控 writer executor seam，Tauri menu builder / tray 事件绑定 / 真实签名 / 公证 / updater 发布链仍未完成。
